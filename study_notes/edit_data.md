# 🎓 14교시: 데이터 수정하기 (Update) 📝

"썼던 글을 고치고 싶어요!"

## 1. 수정의 핵심: 빈 칸 채워두기 (Prefill) 🎨

글쓰기(`Write`)와 수정(`Edit`)의 가장 큰 차이점입니다.
수정 페이지에 들어갔는데 제목과 내용이 비어있으면 황당하겠죠?

1.  수정 페이지(`edit/[id]`)에 들어가면
2.  먼저 **기존 글 내용**을 가져와서 (`GET /posts/1`)
3.  `useState`에 넣어줍니다. (`setTitle("원래 제목")`)
4.  그러면 `input` 태그에 원래 값이 채워져 보입니다! (**Controlled Component**)

## 2. PATCH vs PUT 🥊

데이터를 고칠 때 쓰는 두 가지 방법입니다.

- **PUT**: "이걸로 싹 다 바꿔!" (전체 교체)
- **PATCH**: "이 부분만 고쳐줘!" (부분 수정) 🩹

우리는 제목이나 내용 중 하나만 고칠 수도 있으니 **PATCH**가 더 어울립니다.
(NestJS Controller에도 `@Patch`라고 되어있죠?)

## 3. 구현 전략 🧠

가장 쉬운 방법은 **글쓰기 페이지(`write/page.tsx`)를 복사**해서 만드는 것입니다.
그리고 **두 가지만** 바꾸면 됩니다.

1.  `useEffect`로 데이터 가져오기 (Prefill)
2.  전송 버튼 눌렀을 때 `POST` 대신 `PATCH` 보내기
